> 生成时间：2026-06-07
> 适用版本：当前 master 分支

---

## 一、系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                      服务器端（同一台）                      │
│                                                           │
│  ┌──────────────┐   ┌───────────────┐   ┌──────────┐  │
│  │   PHP 服务    │   │  Python Bridge │   │ Mosquitto │  │
│  │  (Web + API) │──▶│   (port 9100) │   │(port 1883)│  │
│  │              │   │  └──────┬──────┘   │          │  │
│  │              │   │         │           │          │  │
│  │              │   │         ▼           │          │  │
│  │              │   │   火山引擎 TTS API │          │  │
│  └──────┬───────┘   └─────────────────┘   └────┬─────┘  │
│         │                                           │          │
└─────────┼───────────────────────────────────────────┼──────────┘
          │            MQTT (WebSocket)                │
          ▼                                           ▼
  ┌──────────────────────┐                ┌──────────────────────┐
  │   网页浏览器          │                │   ESP32 设备          │
  │   (用户操作入口)     │                │  + MAX98357A + 喇叭  │
  └──────────────────────┘                └──────────────────────┘
```

**数据流向：**

```
用户输入文本
  → PHP 接收请求，自动断句分段
  → PHP 调用 Python Bridge 预生成每段 TTS
  → Python Bridge 调用火山引擎流式 TTS API
  → PHP 通过 MQTT 下发播放命令到 ESP32
  → ESP32 通过 HTTP 拉取音频流
  → ESP32 I2S 输出到 MAX98357A 播放
```

## 二、部署前置检查清单

部署前必须修改以下配置文件中的占位值，否则系统无法运行。

### 2.1 ESP32 固件

**文件：** `arduino/sound/sound.ino`

| 常量 | 说明 | 示例值 |
|------|------|--------|
| `WIFI_SSID` | WiFi 名称 | `"MyWiFi"` |
| `WIFI_PASSWORD` | WiFi 密码 | `"password"` |
| `MQTT_BROKER` | MQTT broker 地址 | `"192.168.1.100"` 或域名 |
| `MQTT_PORT` | MQTT 端口 | `1883` |
| `MQTT_USERNAME` | MQTT 用户名（可选） | `"mqtt_user"` |
| `MQTT_PASSWORD` | MQTT 密码（可选） | `"mqtt_pass"` |
| `DEVICE_ID` | 设备唯一 ID | `"esp32_01"` |

### 2.2 PHP 服务端

**文件：** `php-server/config.php`

| 配置项 | 说明 | 示例值 |
|--------|------|--------|
| `mqtt.host` | MQTT broker 地址（同 ESP32） | `'192.168.1.100'` |
| `mqtt.port` | MQTT 端口 | `1883` |
| `mqtt.username` | MQTT 用户名 | `'mqtt_user'` |
| `mqtt.password` | MQTT 密码 | `'mqtt_pass'` |
| `python_tts.prepare_url` | Python Bridge 地址 | `'http://127.0.0.1:9100/api/tts'` |

> **注意：** PHP 和 Python Bridge 部署在同一台服务器时，`prepare_url` 用 `127.0.0.1` 即可。

### 2.3 Python Bridge

**文件：** `python-vol/tts_bridge_config.py`

| 配置项 | 说明 | 获取方式 |
|--------|------|----------|
| `VOLCENGINE_API_KEY` | 火山引擎 API Key | 火山引擎控制台获取 |
| `VOLCENGINE_RESOURCE_ID` | 资源 ID | 火山引擎控制台获取（如 `seed-tts-2.0`） |
| `VOLCENGINE_SPEAKER` | 发音人 | 如 `zh_female_vv_uranus_bigtts` |
| `TTS_PUBLIC_BASE_URL` | ESP32 可访问的公开地址 | `http://服务器IP:9100` |
| `TTS_BRIDGE_HOST` | 监听地址 | `0.0.0.0`（允许外部访问）|
| `TTS_BRIDGE_PORT` | 监听端口 | `9100` |

> **关键：** `TTS_PUBLIC_BASE_URL` 必须是 ESP32 能通过 WiFi 访问到的地址，不能用 `127.0.0.1`。

## 三、服务器环境准备

### 3.1 系统要求

| 组件 | 最低版本 | 推荐 |
|------|----------|--------|
| OS | Ubuntu 20.04 / CentOS 7 | Ubuntu 22.04 |
| PHP | 7.4 | 8.1 |
| Python | 3.8 | 3.10+ |
| Mosquitto | 1.6 | 2.0+ |
| 内存 | 512MB | 1GB+ |
| 磁盘 | 1GB | 2GB+ |

### 3.2 安装 PHP（Ubuntu/Debian）

```bash
# 添加 PHP PPA（如需新版本）
sudo apt update
sudo apt install -y software-properties-common
sudo add-apt-repository -y ppa:ondrej/php
sudo apt update

# 安装 PHP 8.1 及必要扩展
sudo apt install -y php8.1 php8.1-cli php8.1-common php8.1-curl php8.1-mbstring php8.1-xml

# 验证
php -v
```

### 3.3 安装 PHP（`yum` / CentOS）

```bash
# CentOS 7/8 使用 Remi 仓库
yum install -y epel-release
yum install -y https://rpms.remirepo.net/enterprise/remi-release-7.rpm
yum install -y php81 php81-php-cli php81-php-common php81-php-curl php81-php-mbstring

# 验证
php81 -v
```

### 3.4 安装 Composer

```bash
curl -sS https://getcomposer.org/installer | php
sudo mv composer.phar /usr/local/bin/composer
composer --version
```

### 3.5 安装 Python3 及 pip

```bash
# Ubuntu
sudo apt install -y python3 python3-pip python3-venv

# CentOS
sudo yum install -y python3 python3-pip

# 验证
python3 --version
pip3 --version
```

### 3.6 安装 Mosquitto

```bash
# Ubuntu
sudo apt install -y mosquitto mosquitto-clients
sudo systemctl enable mosquitto
sudo systemctl start mosquitto

# CentOS
sudo yum install -y mosquitto
sudo systemctl enable mosquitto
sudo systemctl start mosquitto

# 验证
mosquitto -v
```

#### Mosquitto 配置（可选密码）

```bash
# 创建密码文件
sudo mosquitto_passwd -c /etc/mosquitto/passwd mqtt_user
# 按提示输入密码

# 编辑配置 /etc/mosquitto/mosquitto.conf，添加：
# allow_anonymous false
# password_file /etc/mosquitto/passwd

sudo systemctl restart mosquitto
```

## 四、Python Bridge 部署

### 4.1 上传/克隆代码

```bash
# 假设部署目录为 /var/www/kai-sound-tts
sudo mkdir -p /var/www
sudo cp -r /path/to/kai-sound-tts /var/www/
sudo chown -R $USER:$USER /var/www/kai-sound-tts
```

### 4.2 安装 Python 依赖

```bash
cd /var/www/kai-sound-tts/python-vol
python3 -m pip install -r requirements.txt
```

### 4.3 配置 `tts_bridge_config.py`

```bash
nano /var/www/kai-sound-tts/python-vol/tts_bridge_config.py
```

按**第二节 2.3** 的说明填写真实值。

### 4.4 手动测试启动

```bash
cd /var/www/kai-sound-tts/python-vol
python3 tts_bridge_server.py --host 0.0.0.0 --port 9100
```

正常输出类似：

```
[2026-06-07 16:00:00] Volcengine TTS bridge running
[2026-06-07 16:00:00] listen: http://0.0.0.0:9100
[2026-06-07 16:00:00] public: http://YOUR_SERVER_IP:9100
```

另开终端测试健康检查：

```bash
curl http://127.0.0.1:9100/health
```

预期返回：

```json
{"ok": true, "service": "volcengine-tts-bridge", "ready": true, ...}
```

### 4.5 注册为 systemd 服务（生产环境推荐）

创建服务文件 `/etc/systemd/system/kai-tts-bridge.service`：

```ini
[Unit]
Description=Kai Sound TTS Python Bridge
After=network.target

[Service]
Type=simple
User=www-data
Group=www-data
WorkingDirectory=/var/www/kai-sound-tts/python-vol
ExecStart=/usr/bin/python3 /var/www/kai-sound-tts/python-vol/tts_bridge_server.py --host 0.0.0.0 --port 9100
Restart=always
RestartSec=5
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable kai-tts-bridge
sudo systemctl start kai-tts-bridge
sudo systemctl status kai-tts-bridge
```

## 五、PHP 服务部署

### 方案 A：宝塔面板部署（推荐，最简单）

1. 登录宝塔面板
2. 点击「网站」→「添加站点」
3. 域名填写服务器 IP 或绑定域名（如 `tts.example.com`）
4. **根目录选择：** `/var/www/kai-sound-tts/php-server/public`
5. PHP 版本选择 7.4 或更高
6. 上传 `php-server/` 整个目录到站点目录的上一级（确保 `config.php` 和 `vendor/` 在正确位置）
7. 进入站点目录，安装 PHP 依赖：

```bash
cd /www/wwwroot/your-site/  # 宝塔实际路径
cd ../
cp -r /var/www/kai-sound-tts/php-server ./php-server
cd php-server
composer install --no-dev
```

8. 在宝塔面板「网站」→ 对应站点 →「设置」→「网站目录」，将运行目录设置为 `php-server/public`

### 方案 B：手动部署（Nginx + PHP-FPM）

#### 安装 Nginx

```bash
# Ubuntu
sudo apt install -y nginx

# CentOS
sudo yum install -y nginx
```

#### 安装 PHP-FPM

```bash
# Ubuntu（延续 3.2 的 PHP 安装）
sudo apt install -y php8.1-fpm

# CentOS
sudo yum install -y php81-php-fpm
```

#### Nginx 站点配置

创建 `/etc/nginx/sites-available/kai-sound-tts`：

```nginx
server {
    listen 80;
    server_name your-domain.com;  # 或服务器 IP

    root /var/www/kai-sound-tts/php-server/public;
    index index.html index.php;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        include snippets/fastcgi-php.conf;
        fastcgi_pass unix:/run/php/php8.1-fpm.sock;  # 按实际版本调整
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        include fastcgi_params;
    }

    # 允许跨域（ESP32 网页控制需要）
    add_header Access-Control-Allow-Origin *;
    add_header Access-Control-Allow-Methods "GET, POST, OPTIONS";
    add_header Access-Control-Allow-Headers "Content-Type";
}
```

```bash
sudo ln -s /etc/nginx/sites-available/kai-sound-tts /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

#### 安装 PHP 依赖

```bash
cd /var/www/kai-sound-tts/php-server
composer install --no-dev
```

#### 配置 `config.php`

```bash
nano /var/www/kai-sound-tts/php-server/config.php
```

按**第二节 2.2** 的说明填写真实值。

## 六、防火墙配置

确保以下端口对外开放：

| 端口 | 用途 | 开放对象 |
|------|------|----------|
| 80/443 | HTTP/HTTPS（Web 控制台） | 所有用户 |
| 9100 | Python Bridge（音频流） | ESP32 所在网段 |
| 1883 | MQTT | PHP 服务 + ESP32 |

```bash
# UFW（Ubuntu）
sudo ufw allow 80/tcp
sudo ufw allow 9100/tcp
sudo ufw allow 1883/tcp
sudo ufw reload

# firewalld（CentOS）
sudo firewall-cmd --permanent --add-port=80/tcp
sudo firewall-cmd --permanent --add-port=9100/tcp
sudo firewall-cmd --permanent --add-port=1883/tcp
sudo firewall-cmd --reload
```

## 七、ESP32 固件烧录

### 7.1 准备 Arduino IDE

1. 安装 [Arduino IDE](https://www.arduino.cc/en/software)（1.8.x 或 2.x）
2. 添加 ESP32 开发板支持：
   - 打开「首选项」→「附加开发板管理器网址」
   - 添加：`https://dl.espressif.com/dl/package_esp32_index.json`
   - 打开「开发板管理器」，搜索 `esp32`，安装

### 7.2 安装依赖库

在 Arduino IDE「库管理器」中安装：

| 库名 | 用途 |
|------|------|
| `PubSubClient` | MQTT 客户端 |
| `ArduinoJson` | JSON 解析 |
| `ESP32-audioI2S` | 音频解码（MP3/WAV）+ I2S 输出 |

### 7.3 修改配置并烧录

1. 打开 `arduino/sound/sound.ino`
2. 按**第二节 2.1** 修改配置
3. 选择开发板：「ESP32 Dev Module」或「ESP32 WROOM DA Module」
4. 选择正确串口号
5. 点击「上传」

### 7.4 硬件连接（MAX98357A I2S 功放）

| MAX98357A 引脚 | ESP32 引脚 | 说明 |
|-----------------|-------------|------|
| VIN | 3.3V 或 5V | 电源 |
| GND | GND | 地 |
| DIN | GPIO 25 | I2S_DOUT |
| BCLK | GPIO 27 | I2S_BCLK |
| LRC | GPIO 26 | I2S_LRC |
| GAIN | 悬空或接 GND | 增益设置 |
| SD | 接 GND | 关闭关断模式 |

喇叭接到 MAX98357A 的 `+` / `-` 输出端。

## 八、启动顺序与验证

### 8.1 标准启动顺序

```bash
# 1. 启动 Mosquitto
sudo systemctl start mosquitto

# 2. 启动 Python Bridge
sudo systemctl start kai-tts-bridge

# 3. 启动 Nginx/PHP-FPM（或宝塔站点）
sudo systemctl start nginx
sudo systemctl start php8.1-fpm

# 4. ESP32 上电（自动连接 WiFi + MQTT）
```

### 8.2 验证步骤

**Step 1：检查 Python Bridge 健康状态**

```bash
curl http://127.0.0.1:9100/health
```

**Step 2：检查 MQTT 连通性**

```bash
# 订阅设备状态主题
mosquitto_sub -h 127.0.0.1 -t "device/+/status" -v
```

给 ESP32 上电，应看到类似输出：

```
device/esp32_01/status {"device_id":"esp32_01","online":true,...}
```

**Step 3：打开 Web 控制台**

浏览器访问 `http://服务器IP/` 或 `http://your-domain.com/`

- 确认「接口状态」显示 `ok`
- 输入测试文本，点击「发送播报」
- ESP32 应开始播放

**Step 4：查看 ESP32 串口日志**

在 Arduino IDE「串口监视器」中（波特率 115200），应看到：

```
[WiFi] OK
[MQTT] OK
[Audio] Playing: http://...
[Audio] MP3 done
```

## 九、生产环境优化建议

### 9.1 Python Bridge 使用 `venv`

```bash
cd /var/www/kai-sound-tts/python-vol
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
# 修改 systemd ExecStart 使用 venv 中的 python
```

### 9.2 Nginx 反向代理 Python Bridge（可选）

将 Python Bridge 隐藏在 Nginx 后面，统一使用 80/443 端口：

```nginx
location /tts/ {
    proxy_pass http://127.0.0.1:9100/tts/;
    proxy_http_version 1.1;
    proxy_set_header Connection "";
}

location /api/tts {
    proxy_pass http://127.0.0.1:9100/api/tts;
    proxy_http_version 1.1;
    proxy_set_header Connection "";
}
```

此时 `TTS_PUBLIC_BASE_URL` 改为 `https://your-domain.com`。

### 9.3 HTTPS

使用 Let's Encrypt 免费证书：

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

### 9.4 日志管理

Python Bridge 日志通过 systemd journal 查看：

```bash
sudo journalctl -u kai-tts-bridge -f
```

PHP 错误日志路径（宝塔）：`/www/wwwlogs/`

## 十、故障排查

| 现象 | 可能原因 | 排查方法 |
|------|----------|----------|
| ESP32 连不上 MQTT | MQTT 地址错误 / broker 未启动 | 检查 `MQTT_BROKER`；`systemctl status mosquitto` |
| 网页点击「发送播报」无反应 | PHP 未运行 / API 报错 | 打开浏览器开发者工具 → Network 面板查看请求 |
| Python Bridge 返回 `missing config` | `VOLCENGINE_API_KEY` 未填写 | 检查 `tts_bridge_config.py` |
| ESP32 播放中断（`stream_interrupted`）| WiFi 信号差 / 路由器问题 | 检查 `WiFi.RSSI()` 数值（应 > -70dBm）|
| 火山引擎返回错误 | API Key 无效 / 额度用完 | 检查 `VOLCENGINE_API_KEY`；查看 Python 终端输出 |
| 段间有停顿 | 正常现象（当前架构限制）| 无需处理，属于设计边界 |
