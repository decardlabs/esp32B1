# 服务端工作原理

> 本文档说明整个 TTS 播报系统中服务端（PHP + Python + Mosquitto）的架构与工作流程。

## 各服务角色

### 1. Nginx (Web 服务)

**端口：80/443**

- 提供静态文件（`index.html` Web 控制台）
- 将 PHP 请求转发给 PHP-FPM 处理
- 站点根目录指向 `server/php/`

### 2. PHP-FPM (API 编排层)

**文件：`server/php/`**

PHP 是整个系统的"大脑"，负责：

1. 接收网页请求
2. 文本断句分段
3. 调用 Python Bridge 预生成 TTS
4. 通过 MQTT 下发播放命令
5. 等待 ESP32 播完一段再发下一段

### 3. Mosquitto (MQTT Broker)

**端口：1883**

- ESP32 订阅 `device/{id}/command` 接收命令
- PHP 通过 MQTT 客户端发布命令
- ESP32 发布 `device/{id}/status` (retained) 上报状态
- 启用密码认证，防止未授权控制
- 遗嘱消息：ESP32 断线时自动发布 `{"online":false}`

### 4. Python Bridge (TTS 代理)

**端口：9100**

- 接收 PHP 的 TTS 合成请求
- 调用 Volcengine 流式 TTS API
- 将合成的音频流以 HTTP 流形式提供给 ESP32
- 每个 TTS 请求有 10 分钟有效期

## PHP 核心模块详解

### tts_request_handler.php

这个文件包含所有 TTS 编排逻辑。

**文本分段函数：**

| 函数 | 作用 |
|------|------|
| `tts_split_text_into_segments()` | 主入口：按句号/感叹号/问号/分号/换行断句，再将短句合并到 70 字，长句按逗号/冒号二次切分，兜底 100 字硬限制 |
| `tts_split_overlong_sentence()` | 按逗号、冒号拆分超长句 |
| `tts_chunk_by_length()` | 按字符数硬切（最后兜底） |

**MQTT 交互函数：**

| 函数 | 作用 |
|------|------|
| `tts_publish_command()` | 向 ESP32 下发 MQTT 命令 |
| `tts_wait_for_segment_finish()` | 订阅 ESP32 状态主题，等待播完（超时 45 秒） |
| `tts_build_mqtt_settings()` | 构建 MQTT 连接参数（用户名密码） |

### device-status.php

通过 MQTT 读取 ESP32 的 retained 状态消息：

```
1. 连接 MQTT Broker (127.0.0.1:1883)
2. 订阅 device/esp32_01/status (QoS 1)
3. 收到 retained 消息 → 返回 online/offline
4. 5 秒超时未收到 → 返回 offline
```

## Python Bridge 详解

### 接口列表

| 路径 | 方法 | 说明 |
|------|------|------|
| `/api/tts` | POST | 创建 TTS 任务，返回 `stream_url` |
| `/tts/{request_id}.{format}` | GET | ESP32 拉取音频流（流式转发 Volcengine） |
| `/api/tts/{request_id}` | GET | 查询 TTS 任务状态 |
| `/health` | GET | 健康检查 |

### TTS 任务生命周期

```
POST /api/tts
  → 生成 request_id (UUID)
  → 存入 TtsRequestStore（内存中，10分钟TTL）
  → 返回 stream_url
  → ESP32 请求 stream_url 时，才开始调用 Volcengine API
  → Volcengine 返回 SSE 流，逐 chunk base64 解码后转发给 ESP32
  → 完成后更新任务状态为 "finished"
```

### 并发处理

- 使用 `ThreadingHTTPServer` 处理并发请求
- `TtsRequestStore` 使用 `threading.Lock` 保护共享数据
- 每个音频流用 `threading.Condition` 实现生产者-消费者模式
- Volcengine 请求和 HTTP 响应在独立线程中并行处理

## MQTT 协议

### 主题约定

| 主题 | 方向 | 说明 |
|------|------|------|
| `device/{id}/command` | PHP → ESP32 | 播放控制命令（QoS 1） |
| `device/{id}/status` | ESP32 → PHP | 状态上报（retained，含遗嘱） |
| `device/{id}/heartbeat` | ESP32 → PHP | 心跳（10 秒间隔） |

### 命令格式

所有命令以 JSON 格式发布到 `device/{id}/command`：

```json
// TTS 播报
{"cmd": "tts", "url": "http://...wav", "text_preview": "你好", "format": "wav"}

// 播放 URL
{"cmd": "play", "params": {"url": "http://..."}}

// 控制
{"cmd": "pause"}
{"cmd": "resume"}
{"cmd": "stop"}

// 音量
{"cmd": "volume", "params": {"value": 80}}

// 测试音
{"cmd": "tone", "params": {"freq": 1000, "duration": 1200}}

// 刷新状态
{"cmd": "status"}
```

### 状态格式

ESP32 上报到 `device/{id}/status`：

```json
{
  "device_id": "esp32_01",
  "online": true,
  "ip": "192.168.x.x",
  "wifi_rssi": -43,
  "volume": 100,
  "play_state": "playing",
  "free_heap": 8580588
}
```

`play_state` 取值：`idle` / `playing` / `paused` / `error`

## ESP32 端播放流程

```
MQTT 收到 cmd=tts {url: "http://...wav"}
  → 停止当前播放
  → HTTP GET 音频流 URL
  → 读取前 256 bytes → 解析 WAV 头 (RIFF/WAVE/data)
  → 获取 sample_rate 和 data_offset
  → 配置 I2S (Philips, 16bit, mono, 对应 sample_rate)
  → 启动 I2S DMA
  → ES8388 取消静音 → 喇叭出声
  → 循环读取 HTTP → 写入 32KB 环形缓冲区 → 解码线程读取 → I2S
  → 流读完 → ES8388 静音 → 状态跳回 idle
```

## 部署配置

### 服务器间通信

| 通信 | 地址 | 说明 |
|------|------|------|
| PHP → Mosquitto | `127.0.0.1:1883` | 同机，走本地 |
| PHP → Python | `127.0.0.1:9100` | 同机，走本地 |
| ESP32 → Mosquitto | `服务器公网IP:1883` | 走公网，需密码认证 |
| ESP32 → Python | `服务器公网IP:9100` | 走公网，拉取音频流 |

### Mosquitto 配置要点

```
listener 1883 0.0.0.0        # 监听公网
allow_anonymous false          # 必须密码
password_file /etc/mosquitto/passwd
```

### Python Bridge 配置要点

```python
TTS_PUBLIC_BASE_URL = "http://服务器公网IP:9100"  # 必须是 ESP32 能访问的地址
```
